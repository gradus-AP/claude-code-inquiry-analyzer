# CS分析エージェント（統合） — `/cs`

「分析して」と言われたら、以下の3ステップで実行:

1. **主題を選択**させる
2. **期間・仮説をオプション**で設定
3. 結果に応じて `/cs-analyze` または `/cs-churn` を実行

---

## 実行フロー

### Step 1: ユーザーに主題を選ばせる

以下のいずれかを選択させてください：

```
🔹 トピック分析
  └─ 「ログインできない」「機能の使い方」など8つのカテゴリから分析
🔴 解約リスク分析
  └─ MLモデルで全企業の週次スコアリング + 高リスク企業の深掘り
🔧 その他
  └─ 独自のカスタム分析質問
```

### Step 2: オプション設定（期間・仮説）

**期間:**
- 📅 デフォルト（直近28日）
- 📅 カスタム入力（例: 直近90日）

**仮説（オプション）:**
- 🤔 デフォルト（スキルが自動生成）
- 🤔 カスタム仮説を3つ入力（例: 「新規顧客のオンボーディング不足？」）

### Step 3: 分析実行

選択結果に応じて以下を実行：

- **トピック分析** → `/cs-analyze {topic}` を実行
- **解約リスク分析** → `/cs-churn` を実行
- **その他** → ユーザーに詳細を確認

## 実装例（AskUserQuestion 使用）

```yaml
Question 1: 分析の主題は？
  Options:
    - トピック分析（8つのカテゴリから選択）
    - 解約リスク分析（全企業スコアリング）
    - その他（カスタム質問）

Question 2: 分析期間は？
  Options:
    - デフォルト（直近28日）
    - カスタム（例: 90日, 6ヶ月）

Question 3: 仮説設定は？
  Options:
    - スキル自動生成（推奨）
    - 仮説を手入力（最大3つ）
```

## 設計方針

- **オプション優先**: 迷ったときはデフォルト（自動生成）で大丈夫
- **カスタマイズ可能**: 「この仮説を検証したい」という具体的な要望に対応
- **UX改善**: ユーザーが何を分析したいかを明確に意思表示できる
- **アジャイル**: 試行錯誤しながら質問項目を調整
